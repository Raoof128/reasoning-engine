# API Reference

All tools are exposed via the MCP (Model Context Protocol) interface. The server name is `reasoning-engine`.

---

## init_research_session

Create a new research session. Estimates difficulty and returns budget allocation.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `query` | string | Yes | The research question to investigate |

**Returns:**

```json
{
  "session_id": "uuid",
  "query": "the research question",
  "difficulty": 0.71,
  "strategy": "forest",
  "budget": {
    "total_branches": 8,
    "max_depth": 12,
    "max_steps": 50,
    "tokens_per_branch": 5000,
    "remaining_steps": 50
  },
  "status": "active"
}
```

**Strategy mapping:**

| Difficulty | Strategy | Branches | Depth | Steps |
|-----------|----------|----------|-------|-------|
| 0.0 - 0.3 | single_pass | 1 | 3 | 5 |
| 0.3 - 0.5 | best_of_n | 3 | 5 | 15 |
| 0.5 - 0.7 | beam_search | 5 | 8 | 30 |
| 0.7 - 1.0 | forest | 8 | 12 | 50 |

## register_branch

Register a new reasoning branch generated by an Actor agent.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `session_id` | string | Yes | Session this branch belongs to |
| `trace` | string | Yes | JSON array of reasoning steps |
| `sources` | string | No | JSON array of source objects (`[{"url", "title", "excerpt"}]`). Default: `"[]"` |
| `parent_id` | string | No | Parent branch ID for tree structure |
| `depth` | integer | No | Depth in the search tree. Default: `0` |

**Returns:**

```json
{
  "branch_id": "uuid",
  "session_id": "uuid"
}
```

## score_branch

Record a Critic's dual-signal score for a reasoning branch.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `session_id` | string | Yes | Session ID |
| `branch_id` | string | Yes | Branch being scored |
| `q_score` | number | Yes | Promise: expected probability of success (0-1) |
| `advantage` | number | Yes | Progress: improvement over baseline (0-1) |
| `critique` | string | Yes | Textual critique (strengths, weaknesses, gaps) |
| `confidence` | number | Yes | Critic's self-assessed confidence (0-1) |

**Returns:**

```json
{
  "status": "scored",
  "branch_id": "uuid"
}
```

## select_next_branches

Run DORA allocation. Computes kappa (score variance), decides explore vs exploit, returns branch decisions.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `session_id` | string | Yes | Session to evaluate |

**Returns:**

```json
{
  "branches_to_continue": ["uuid1", "uuid2"],
  "branches_to_reflect": ["uuid3"],
  "branches_to_prune": ["uuid4"],
  "allocation": "breadth",
  "kappa": 0.65,
  "budget_remaining": 42
}
```

**Allocation logic:**

- `kappa > 0.15`: Breadth (explore). Scores are spread. Keep multiple branches.
- `kappa < 0.15`: Depth (exploit). Scores converge. Focus on top 1-2.
- Branches with Q-score < 0.25 are pruned.
- Branches with Q-score < 0.50 are sent to Reflexion.

## check_termination

Check whether the research session should terminate.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `session_id` | string | Yes | Session to check |

**Returns:**

```json
{
  "should_terminate": true,
  "reason": "High confidence result (q=0.91, conf=0.88)"
}
```

**Termination conditions:**

1. Budget exhausted (remaining_steps <= 0)
2. Top branch Q-score > 0.85 AND confidence > 0.80
3. Otherwise: continue

## consensus_candidates

Return top-K branches by Q-score for final synthesis.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `session_id` | string | Yes | Session ID |
| `top_k` | integer | No | Number of top branches. Default: `3` |

**Returns:**

JSON array of branch objects (same schema as database rows), sorted by Q-score descending.

## record_reflection_tool

Store a Reflexion cycle's critique and revision. Re-activates the branch after reflection.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `session_id` | string | Yes | Session ID |
| `branch_id` | string | Yes | Branch that was reflected on |
| `original_critique` | string | Yes | The Critic's textual critique |
| `revision_summary` | string | Yes | Summary of revisions made |
| `score_before` | number | Yes | Q-score before reflection |
| `score_after` | number | Yes | Q-score after reflection |

**Returns:**

```json
{
  "id": "uuid",
  "score_before": 0.4,
  "score_after": 0.7
}
```

## save_to_memory

Persist episodic memory from a research session for future recall.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `session_id` | string | Yes | Source session |
| `query` | string | Yes | The research query |
| `key_learnings` | string | Yes | JSON array of learning strings |
| `domain_tags` | string | Yes | JSON array of domain tag strings |

**Returns:**

```json
{
  "id": "uuid"
}
```

---

## recall_memory_tool

Retrieve relevant episodic memories from past research sessions. Uses keyword overlap scoring.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `query` | string | Yes | Query to search memories for |
| `limit` | integer | No | Max results. Default: `5` |

**Returns:**

JSON array of memory objects with `query`, `key_learnings`, `domain_tags`, `created_at`.

---

## sanitize_content

Strip HTML, script tags, and prompt injection patterns from web-crawled content.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `raw_text` | string | Yes | Raw text to sanitize |

**Returns:**

```json
{
  "cleaned": "sanitized text content"
}
```

**Patterns removed:** `<script>`, `<style>`, all HTML tags, "ignore all previous instructions", "system: override", chat template markers (`[INST]`, `<<SYS>>`).

---

## get_session_state

Return full session state including all active branches and scores. Useful for debugging.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `session_id` | string | Yes | Session to inspect |

**Returns:**

```json
{
  "session": { "id": "...", "query": "...", "difficulty": 0.71, ... },
  "active_branches": [ { "id": "...", "q_score": 0.85, ... } ]
}
```
